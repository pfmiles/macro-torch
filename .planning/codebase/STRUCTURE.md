## Directory Purposes

This is a flat-structure project with no Lua subdirectories. All source files live at the project root. This is intentional -- the build script uses `find ./ -iname '*.lua'` and files are concatenated in a specific order.

**Build output:**
- Purpose: Generated SuperMacro addon file deployed to game directory
- Contains: Concatenation of all source `.lua` files in build order
- Key files: `SM_Extend.lua` (gitignored, generated by `build.sh`)

**Projects skills/configuration:**
- `.claude/`: Claude Code skill definitions
- `.claude-reference/`: Reference documentation (WoW API functions)

**Planning artifacts:**
- `.planning/`: GSD planning documents, codebase maps

## Build Concatenation Order

The `build.sh` script produces `SM_Extend.lua` by concatenating files in this exact order:

1. `macro_torch.lua` -- Global namespace initialization (`macroTorch = {}`)
2. `impl_util.lua` -- Core utility functions (must be available before anything else)
3. `interface_debug.lua` -- Debug/show functions (used throughout)
4. `Unit.lua` -- Base Unit class (parent of all other classes)
5. All remaining `*.lua` files (excluding `SM_Extend.lua` and the first 4)

This order ensures that dependency relationships are satisfied: utilities before classes, base class before subclasses, event system before rotation logic.

## Naming Conventions

**Files:**
- `PascalCase.lua`: Class files (Unit.lua, Player.lua, Target.lua, Pet.lua)
- `snake_case.lua`: Utility and system files (impl_util.lua, battle_event_queue.lua, texture_map.lua, event_stack.lua)
- `SM_Extend_{Class}.lua`: Class-specific logic files (prefix refers to "SuperMacro Extend")

**Directories:**
- No Lua subdirectories. Flat project structure. All `.lua` files at root.

**Namespaces:**
- All code in `macroTorch.*` global namespace
- Class constructors: `macroTorch.{ClassName}:new()`
- Singleton instances: `macroTorch.{className}` (lowercase)

**Functions:**
- `camelCase`: Instance methods (`hasBuff`, `targetEnemy`, `startAutoAtk`)
- `camelCase`: Module functions (`catAtk`, `bearAtk`, `hunterAtk`, `rogueAtk`)
- `camelCase`: Utility functions (`isSpellCooledDown`, `getItemBagIdAndSlot`, `computeClaw_E`)
- `safeCamelCase` / `readyCamelCase`: Dual-pattern ability functions (`safeShred`/`readyShred`, `safeClaw`/`readyClaw`)
- `is/Has/has` prefix: Boolean-returning predicates (`isFormActive`, `isSpellReady`, `hasBuff`, `hasItem`)

**Variables:**
- `camelCase`: Local variables (`clickContext`, `lastCast`, `energyDuringGcd`)
- `UPPER_SNAKE_CASE`: Constants and energy costs (`KS_CP1_Health`, `RIP_BASE_DURATION`, `CLAW_E`, `COWER_E`, `DEBUFF_LAND_LAG`)
- `UPPER_SNAKE_CASE`: Field function maps (`UNIT_FIELD_FUNC_MAP`, `PLAYER_FIELD_FUNC_MAP`, `DRUID_FIELD_FUNC_MAP`)

**Types/Tables:**
- `PascalCase`: Class prototypes (`macroTorch.Unit`, `macroTorch.Player`, `macroTorch.Druid`)

## Where to Add New Code

**New Feature (e.g., new class ability):**
- Add helper functions in the appropriate `SM_Extend_{Class}.lua` file
- If shared across classes, add to `biz_util.lua` (spell/item utilities) or `impl_util.lua` (general utilities)
- Follow the `safeXxx`/`readyXxx` dual-pattern for ability functions
- Add any new texture mappings to `texture_map.lua`

**New Class:**
- Create `SM_Extend_{NewClass}.lua` at project root
- Follow the existing pattern: inherit from `macroTorch.Player:new()`, define class constructor with `FIELD_FUNC_MAP`, create singleton instance
- Register any spell tracing via `macroTorch.setSpellTracing()` and `macroTorch.setTraceSpellImmune()`
- Add entry point function (e.g., `macroTorch.warriorAtk()`)
- The build script auto-includes new `.lua` files (no build config changes needed)

**New Component/Module (e.g., new combat subsystem):**
- Create a new `.lua` file at project root with `snake_case.lua` naming
- All functions must be under `macroTorch.*` namespace
- If the file needs to be loaded before certain class files, update `build.sh` to explicitly concatenate it in the correct position

**Utilities:**
- General-purpose utilities (strings, tables, type checks): `impl_util.lua`
- Business logic utilities (spells, items, talents, equipment): `biz_util.lua`
- Debug/display functions: `interface_debug.lua`

**Field Function Maps (new computed property):**
- Add entry to the appropriate `*_FIELD_FUNC_MAP` table in the class's Lua file
- Format: `['propertyName'] = function(self) return ... end`
