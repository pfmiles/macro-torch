# Quick Task 260620-j2p: Opener & Mana Potion Level-Adaptive Fix

**Created:** 2026-06-20
**Mode:** quick
**Goal:** 将 catAtk 中两处60级硬编码改为 level-adaptive 逻辑

## Background

分析发现 catAtk 有两个在低等级实战中会出问题的硬编码：
1. Opener 模块 1500 血量阈值 — 低等级 Pounce/Ravage 伤害远低于60级
2. `humanFormMana < 350` — 绝对数值不适配低法力池 + 猫形态下 UnitMana 返回能量的语义问题

## Task 1: Opener Health Threshold → Level-Adaptive Lookup

**文件:** `classes/druid/Druid.lua` (新增函数), `classes/druid/combo.lua` (替换硬编码)

**改动:**
1. 在 `Druid.lua` 中新增 `macroTorch.getOpenerHealthThreshold(level)` 函数，参照已有的 `estimatePlayerDPS` 和 `getKSThreshold` 的 level-adaptive lookup table 模式：
   - 60级: 1500（向后兼容）
   - 50-59: 1000
   - 40-49: 600
   - 30-39: 300
   - <30: 150
2. 在 `combo.lua:114` 将 `target.health >= 1500` 替换为 `target.health >= macroTorch.getOpenerHealthThreshold()`

## Task 2: Mana Potion Threshold → Percentage-Based

**文件:** `classes/druid/Druid.lua` (新增函数), `classes/druid/combo.lua` (替换硬编码)

**问题分析:**
- `humanFormMana` 在猫形态下 `UnitMana('player')` 返回能量(0-100)，不是法力
- 即使在人形态，低等级法力池500-800，350阈值意味着法力低于70%-44%就喝药，过于激进

**改动:**
1. 在 `Druid.lua` 中新增 `macroTorch.shouldUseManaPotion()` 函数：
   - 使用 `UnitMaxMana('player')` 获取最大法力（所有形态下都正确）
   - 阈值设为最大法力的30%（适配所有等级）
   - 猫/熊形态下：`UnitMana('player')` 返回能量/怒气，无法获取真实法力值。但战斗中药水有CD（2分钟），实际不会每场战斗都喝。百分比阈值+药水CD的组合已经足够防止浪费。
   - 人形态下：`UnitMana('player')` 正常返回法力值 → 百分比判断正确
2. 新增辅助 `macroTorch.getManaPotionThreshold()` 返回 `UnitMaxMana('player') * 0.3`
3. 在 `combo.lua:95` 将 `player.humanFormMana < 350` 替换为 `macroTorch.shouldUseManaPotion()`

**注意:** 猫形态下 `UnitMana` 返回能量的 WoW API 行为不在此次修复范围内（需要额外的法力追踪基础设施）。百分比阈值已确保：（a）低等级时阈值随法力池缩放；（b）药水CD防止连续浪费。
