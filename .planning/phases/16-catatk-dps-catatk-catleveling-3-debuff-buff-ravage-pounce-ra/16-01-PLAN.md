将 classes/druid/leveling.lua 从现有 ~77 行骨架完全替换为完整的 catLeveling() 实现，按以下结构编写：

**函数签名:** function macroTorch.catLeveling() — 无参数，所有战斗按正常模式处理

**步骤1 — 形态和目标守卫（保留现有逻辑）：**
- 非猫形态 → return
- 目标不可攻击 → targetEnemy() → return

**步骤2 — 简化 clickContext 构建（D-01/D-08，无 rough/ERPS/reshift/relic 字段）：**
仅包含以下字段：
- 能量消耗：POUNCE_E=50, CLAW_E=macroTorch.computeClaw_E(), SHRED_E=macroTorch.computeShred_E(), RAKE_E=macroTorch.computeRake_E(), BITE_E=35, RIP_E=30, TIGER_E=macroTorch.computeTiger_E()
- 持续时间：TIGER_DURATION=macroTorch.computeTiger_Duration()
- 状态快照：prowling=player.isProwling, comboPoints=player.comboPoints, ooc=player.isOoc, isBehind=target.isCanAttack and player.isBehindTarget
- 免疫标记：isImmuneRake=target.isImmune('Rake'), isImmuneRip=target.isImmune('Rip')

明确排除的字段（不设置）：rough, AUTO_TICK_ERPS, TIGER_ERPS, RAKE_ERPS, RIP_ERPS, POUNCE_ERPS, BERSERK_ERPS, hasEssenceOfTheRed, RESHIFT_ENERGY, normalRelic, FF_DURATION

**步骤3 — 起手技模块（REQ-16-01）：**
在 clickContext.prowling 时执行，复用 isTrivialBattleOrPvp 和 getOpenerHealthThreshold：
- isSpellExist('Pounce') 且 Pounce 可用 + 非免疫 + 血量>=阈值 + 非快速战斗 → player.pounce() → return
- 否则 isSpellExist('Ravage') → player.ravage('ready') → return

**步骤4 — 自动攻击：**
在战斗中（非潜行阶段）：player.startAutoAtk()

**步骤5 — 斩杀优先模块（REQ-16-03, D-05, D-07）：**
所有后续模块执行前！先检查：
- isKillShotOrLastChance(clickContext) 为 true
- 且 comboPoints > 0
- 且 isSpellExist('Ferocious Bite')
→ player.ferocious_bite('raw') → return

**步骤6 — 猛虎之怒模块（REQ-16-02, D-02）：**
- isSpellExist("Tiger's Fury") guard
- isTigerPresent(clickContext) 为 false
- 且 target.distance <= 20
- 且 tigerSelfGCD(clickContext) == 0
- 且 player.mana >= clickContext.TIGER_E
→ player.tigers_fury('ready') → 记录 macroTorch.loginContext.tigerTimer = GetTime() → return

**步骤7 — Rip 模块（REQ-16-02, D-02）：**
- isSpellExist('Rip') guard
- 且 shouldCastRip(clickContext) 为 true
- 且 isGcdOk(clickContext) 且 isNearBy(clickContext)
→ player.rip() → return

**步骤8 — Rake 模块（REQ-16-02, D-02）：**
- isSpellExist('Rake') guard
- isRakePresent(clickContext) 为 false
- 且 not clickContext.isImmuneRake （目标不免疫流血）
- 且 isGcdOk(clickContext) 且 isNearBy(clickContext)
- 且 comboPoints < 5 （5星不打 Rake）
→ player.rake() → return

**步骤9 — 精灵之火模块（REQ-16-02, D-02）— 简化内联版：**
- isSpellExist('Faerie Fire (Feral)') guard
- 非 ooc + 非免疫 FF + player.isSpellReady('Faerie Fire (Feral)') + isGcdOk
- 核心逻辑：当前能量低于 Claw 消耗（即"无事可做等能量"时），用 FF 作为填充技
  → player.mana < clickContext.CLAW_E 则 player.faerie_fire_feral('raw') → return
- 不调用 shouldCastFFDuringWaitWindow（其内部调用 computeErps）

**步骤10 — Bite 终结技模块（D-05）：**
- isSpellExist('Ferocious Bite') guard
- 分支：
  a) OOC 触发 + comboPoints > 0 → ferocious_bite('ready') → return
  b) shouldUseBite(clickContext) 为 true 且 player.mana >= clickContext.BITE_E → ferocious_bite() → return

**步骤11 — 攒星技模块（REQ-16-02, D-02, D-04）— 简化内联版：**
- isFightStarted(clickContext) 且 comboPoints < 5
- 不调用 shouldUseShred（其内部调用 computeErps）。使用简化决策：
  - a) 非 OOC 时仅在有足够能量时攻击（player.mana >= clickContext.CLAW_E 或 >= clickContext.SHRED_E）
  - b) 选择 Shred 的条件：isSpellExist('Shred') + isBehind + 未刚失败背后攻击 → OOC? shred('ready') : shred()
  - c) 否则选择 Claw：isSpellExist('Claw') → OOC? claw('ready') : claw()
  - d) OOC 时必须有一个可用技能即可释放（不检查能量）

**关键实现约束：**
- 每个模块第一个成功动作后必须 return
- 所有技能调用使用点号语法（player.claw()）
- 不使用 # 长度运算符
- 不调用任何 idol/relic 函数（D-08）
- 不调用 computeErps, shouldUseShred, shouldCastFFDuringWaitWindow, shouldDoReshift, computeReshiftEnergy, getMinimumAffordableAbilityCost
- 不调用 catAtk 的 keep 型模块（keepRip, keepRake, keepTigerFury, keepFF, regularAttack）
- 不包含 reshift 模块
- 删除现有 <24 分支
- 保持 Apache 2.0 License 头部注释
