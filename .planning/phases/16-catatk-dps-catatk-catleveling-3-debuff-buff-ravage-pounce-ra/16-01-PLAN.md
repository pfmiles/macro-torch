**函数签名（D-01）:** function macroTorch.catLeveling(rough)

**步骤1 — 形态和目标守卫（保留现有逻辑）：**
- 非猫形态 → return
- 目标不可攻击 → targetEnemy() → return

**步骤2 — clickContext 构建（D-01，简化版，不含relic字段）：**
按 RESEARCH.md:437-474 的模式构建，字段包括：
- 能量消耗：POUNCE_E=50, CLAW_E=macroTorch.computeClaw_E(), SHRED_E=macroTorch.computeShred_E(), RAKE_E=macroTorch.computeRake_E(), BITE_E=35, RIP_E=30, TIGER_E=macroTorch.computeTiger_E()
- 持续时间：TIGER_DURATION=macroTorch.computeTiger_Duration(), FF_DURATION=40
- ERPS 字段（供 computeErps 使用）：AUTO_TICK_ERPS=20/2, TIGER_ERPS=10/3, RAKE_ERPS=macroTorch.computeRake_Erps(), RIP_ERPS=macroTorch.computeRip_Erps(), POUNCE_ERPS=0, BERSERK_ERPS=0, hasEssenceOfTheRed=false
- 状态快照：rough=macroTorch.toBoolean(rough), prowling=player.isProwling, comboPoints=player.comboPoints, ooc=player.isOoc, isBehind=target.isCanAttack and player.isBehindTarget
- 免疫标记：isImmuneRake=target.isImmune('Rake'), isImmuneRip=target.isImmune('Rip')
- Reshift能量：RESHIFT_ENERGY=macroTorch.computeReshiftEnergy()

注意：RAKE_ERPS 和 RIP_ERPS 使用动态函数而非硬编码 0，使 Ancient Brutality 天赋（~30级+）能正确生效。

**步骤3 — 起手技模块（REQ-16-01, D-01）：**
在 clickContext.prowling 时执行，复用 isTrivialBattleOrPvp 和 getOpenerHealthThreshold：
- isSpellExist('Pounce') 且 Pounce 可用 + 非免疫 + 血量>=阈值 + 非快速战斗 → player.pounce() → return
- 否则 isSpellExist('Ravage') → player.ravage('ready') → return

**步骤4 — 自动攻击：**
在战斗中且已结束潜行起手阶段后：player.startAutoAtk()

**步骤5 — 斩杀优先模块（REQ-16-03, D-05, D-07）：**
所有后续模块执行前！先检查：
- isKillShotOrLastChance(clickContext) 为 true
- 且 comboPoints > 0
- 且 isSpellExist('Ferocious Bite')
→ player.ferocious_bite('raw') → return

**步骤6 — 猛虎之怒模块（REQ-16-02, D-02）：**
- isSpellExist("Tiger's Fury") guard
- isTigerPresent(clickContext) 为 false
- 且 target.distance <= 20
- 且 tigerSelfGCD(clickContext) == 0
- 且 player.mana >= clickContext.TIGER_E
→ player.tigers_fury('ready') → 记录 macroTorch.loginContext.tigerTimer = GetTime() → return

**步骤7 — Rip 模块（REQ-16-02, D-02）：**
- isSpellExist('Rip') guard
- 且 shouldCastRip(clickContext) 为 true
- 且 isGcdOk(clickContext) 且 isNearBy(clickContext)
→ player.rip() → return

**步骤8 — Rake 模块（REQ-16-02, D-02）：**
- isSpellExist('Rake') guard
- isRakePresent(clickContext) 为 false（注意：不是Rip，Rake是保持其存在）
- 且 isGcdOk(clickContext) 且 isNearBy(clickContext)
→ player.rake() → return

**步骤9 — 精灵之火模块（REQ-16-02, D-02）：**
- isSpellExist('Faerie Fire (Feral)') guard
- 复用 shouldCastFFDuringWaitWindow(clickContext) — 已在 Druid.lua 中定义，仅依赖 clickContext 能量字段，零成本调用
- 且 isGcdOk(clickContext)
→ player.faerie_fire_feral('raw') → return

**步骤10 — Bite 终结技模块（D-05）：**
- isSpellExist('Ferocious Bite') guard
- 分支：
  a) OOC 触发 + comboPoints > 0 → ferocious_bite('ready') → return
  b) shouldUseBite(clickContext) 为 true 且 player.mana >= clickContext.BITE_E → ferocious_bite() → return
- 注意：此处唤出的bite是普通bite（非raw模式），因为斩杀已在步骤5优先处理

**步骤11 — 攒星技模块（REQ-16-02, D-02, D-04）：**
- isFightStarted(clickContext) 且 comboPoints < 5
- shouldUseShred(clickContext) 判断 Shred vs Claw：
  a) 选择 Shred: isSpellExist('Shred') → OOC? shred('ready') : shred() → return
  b) 选择 Claw: isSpellExist('Claw') → OOC? claw('ready') : claw() → return

**步骤12 — Reshift 模块（D-03）：**
- isSpellExist('Reshift', 'spell') guard
- 调用 shouldDoReshift(clickContext)（共享函数，Druid.lua中定义，首行检查 RESHIFT_ENERGY==0 → 自动跳过低等级）
- 若返回 true: 调用 macroTorch.readyReshift(clickContext, nextMove, minAbilityCost)（cat.lua中已定义，复用即可）
→ return

**关键实现约束：**
- 每个模块第一个成功动作后必须 return（一键宏一次一个动作）
- 所有技能调用使用点号语法（player.claw()，不是 player:claw()）
- 不使用 # 长度运算符
- 不调用任何 idol/relic 相关函数（computeNormalRelic, recoverNormalRelic, dischargeEnergyChangeRelicAndRip）
- 不调用 catAtk 的 keep 型模块（keepRip, keepRake, keepTigerFury, keepFF, regularAttack）
- 删除现有 <24 分支和注释掉的未来扩展代码块，由 isSpellExist guard 统一处理等级差异
- 保持 Apache 2.0 License 头部注释
