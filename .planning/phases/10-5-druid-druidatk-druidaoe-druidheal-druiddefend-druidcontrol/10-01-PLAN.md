<objective>
Create `classes/druid/combo.lua` containing 5 global top-level "combo" macro methods that route to form-specific sub-methods via if-elseif chains.

Purpose: Provide one-button macro bindings. Users bind a single macro key per action type (attack, AOE, heal, defend, control). Each method detects the player's current shapeshift form and routes to the appropriate sub-method automatically.

Output: `classes/druid/combo.lua` with `macroTorch.druidAtk`, `macroTorch.druidAoe`, `macroTorch.druidHeal`, `macroTorch.druidDefend`, `macroTorch.druidControl`

    **macroTorch.druidAoe() — per D-06/D-07/D-08:**
    - Bear branch (`player.isInBearForm`): call `macroTorch.bearAoe()` (global function, bear.lua:64)
    - Humanoid/caster branch (`not player.isInCatForm and not player.isInBearForm`): check mana (per Claude's discretion from RESEARCH.md edge case 5 — use `player.mana >= 880`), then call `macroTorch.player.hurricane('ready')`. If insufficient mana, return.

    - No auto-form-switch (per D-02)
    - No clickContext needed for hurricane — just a simple guard + cast

    **macroTorch.druidHeal() — per D-09/D-10/D-11/D-12:**
    - One action per keypress (per D-09 — each press does exactly one thing)
    - Step 1: If in cat or bear form (`player.isInCatForm or player.isInBearForm`), call `CancelShapeshiftForm()` and return. Use this inline check, NOT `isInCasterForm` (which only detects Moonkin, per D-11/RESEARCH.md).
    - Step 2 (falls through after form is cancelled on previous press): Check if Rejuvenation is ready, player HP < 50% (`player.healthPercent < 50`), and player does not already have Rejuvenation HOT. For HOT check, use `player.hasBuff('Spell_Nature_Rejuvenation')` (per Claude's discretion in RESEARCH.md edge case 3). If all true, cast `macroTorch.player.rejuvenation('safe', true)` and return.
    - Step 3: If player HP < 40% (`player.healthPercent < 40`), cast `macroTorch.player.healing_touch('safe', true)` and return.
    - Step 4: Otherwise do nothing (return)
    - V1 self-heal only (onSelf=true, per D-10)
    - Use `macroTorch.player.healthPercent` (from Unit.lua UNIT_FIELD_FUNC_MAP), not raw `UnitHealth('player') / UnitHealthMax('player')`

    **Coding conventions:**
    - Dot syntax only (no colon syntax). Use `macroTorch.player.catAtk(rough)` not `macroTorch.player:catAtk(rough)`.
    - Do NOT use `#` unary length operator.
    - Do NOT set `ref` directly on combo methods — ref is inherited through the metatable chain.
    - Functions are global `function macroTorch.druidAtk(rough)` style, matching bear.lua/bearAtk pattern.

    - Merge the logic from old druidStun (utility.lua:13-32) and old druidControl (utility.lua:49-56)
    - Create `local clickContext = {}` for distance caching (following cat.lua/bear.lua pattern)
    - Guard: create `local target = macroTorch.target`. If `not target.isCanAttack`, try `macroTorch.player.targetEnemy()`. If still no valid target, return.
    - Bear form branch (`player.isInBearForm`):
      a. If melee range (`macroTorch.isNearBy(clickContext)` — same pattern as utility.lua:25), cast `macroTorch.player.bash('ready')` and return

    - Humanoid form branch (`not player.isInCatForm and not player.isInBearForm`):
      a. If target is Beast or Dragonkin (`target.type == 'Beast' or target.type == 'Dragonkin'`), cast `macroTorch.player.hibernate('safe')` and return
      b. Otherwise, cast `macroTorch.player.entangling_roots('safe')` and return
    - Other forms (cat/travel/aquatic): auto-shift. If in cat/travel/aquatic form, cast `macroTorch.player.dire_bear_form('ready')` to shift to bear first, return (next press enters bear branch). This follows D-16's "non-bear non-humanoid -> auto-shift to bear or humanoid" with the pragmatic choice of bear (matches old druidStun pattern at utility.lua:18-20).
