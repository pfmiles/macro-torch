**After — 5 个声明式注册 (per RESEARCH Code Examples):**
```lua
-- tracing and immune registration for druid spells (declarative style)
macroTorch.SpellTrace:register('Pounce', {
    spellId = 9827, land = true,
    immune = true, debuffTexture = 'Ability_Druid_SupriseAttack'
})
macroTorch.SpellTrace:register('Rake', {
    spellId = 9904, land = true,
    immune = true, debuffTexture = 'Ability_Druid_Disembowel'
})
macroTorch.SpellTrace:register('Rip', {
    spellId = 9896, land = true,
    immune = true, debuffTexture = 'Ability_GhoulFrenzy'
})
macroTorch.SpellTrace:register('Ferocious Bite', {
    spellId = 31018, land = true,
    immune = false  -- FB has consumeLandEvent but NO immune tracing
})
macroTorch.SpellTrace:register('Faerie Fire (Feral)', {
    land = false,  -- FF is not spell-traced (no setSpellTracing call in original)
    immune = true, debuffTexture = 'Spell_Nature_FaerieFire'
})
```

**关键注意事项:**
- Ferocious Bite 原始代码只有 `setSpellTracing` 没有 `setTraceSpellImmune`，所以 `immune = false`
- Faerie Fire (Feral) 原始代码只有 `setTraceSpellImmune` 没有 `setSpellTracing`，所以 `land = false`
- spellId 字段直接使用原始硬编码数值 (per RESEARCH A3 — addon 加载期间 GetSpellName 不可用)
- 不添加 Shred/Claw 的注册 (当前代码没有追踪它们，遵循现有行为不扩展范围)
- 注释保留原始注释的精神: `-- tracing certain spells and maintain the landTable` + `-- register druid spells immune tracing`

**不修改:** 其他任何 Druid 代码、猫形态逻辑、能量常量、函数定义。仅替换第 479-489 行。

F1. 猫形态技能存在性 (10 项):

- 必须覆盖: Shred, Rip, Rake, Claw, Ferocious Bite, Tiger's Fury, Faerie Fire, Pounce, Ravage, Cower
- 测试命名: `"Druid: Shred() exists"` 格式

F2. Talent 等级检测 (5 项):
- 使用 `macroTorch.player.talentRank(tabIndex, talentIndex)` 验证 talent 等级 >= 1 (如果玩家点了该 talent)
- 必须覆盖: Ancient Brutality, Omen of Clarity, Ferocity, Improved Shred, Blood Frenzy
- 注意: 玩家可能没有点某个 talent，此时 `talentRank` 返回 0。对于 "talent exists" 类型的检查，验证 `type(talentRank(...)) == "number"` 即可
- 不使用断言特定等级值 — 仅验证返回值为数字 (存在性 + 可调用性)
- 测试命名: `"Druid: Ancient Brutality talent rank >= 0"` 格式

F3. 能量常量范围验证 (3 项):
- 验证常量取值在合理范围内:
  - `CLAW_E` 范围 40-50 (基值 45)
  - `SHRED_E` 范围 48-60 (基值 60)
  - `RAKE_E` 范围 35-40 (基值 40)
- 测试命名: `"Druid: CLAW_E in range"` 格式
- 验证宏: `assert(CLAW_E >= 35 and CLAW_E <= 55, "CLAW_E = " .. tostring(CLAW_E))`

**类别 G: Druid optional 检测 (>=7 项, isOptional=true) per D-09:**

G1. DRUID_FIELD_FUNC_MAP 字段完整性 (5 项):
- 验证 `macroTorch.player` 上有以下属性且 type 正确:
  - comboPoints: `type(macroTorch.player.comboPoints) == "number"`
  - isOoc: `type(macroTorch.player.isOoc) ~= "nil"` (通过调用确定性)
  - isProwling: `type(macroTorch.player.isProwling) == "boolean"` via toBoolean
  - isBerserk: `type(macroTorch.player.isBerserk) ~= "nil"`
  - humanFormMana: `type(macroTorch.player.humanFormMana) ~= "nil"` (仅在允许情况下调用)
- 测试命名: `"Druid: DRUID_FIELD_FUNC_MAP comboPoints exists"` 格式
- isOptional=true — 这些是 field func map 完整性检测，不影响核心功能

G2. 形态检测 (2 项):
- `type(macroTorch.player.isCatForm) == "boolean"` via toBoolean — 验证属性存在
- `type(macroTorch.player.isBearForm) == "boolean"` via toBoolean — 同上
- isOptional=true — 形态切换由游戏运行时决定，自检时仅验证属性存在性
- 测试命名: `"Druid: isCatForm exists"` 格式

macroTorch.SelfTest:register("Druid: Shred() exists", function()
